# [로또축구단] 일일 개발 로그 생성 지침서 (CLI용)

이 문서는 CLI 도구가 매일 개발 로그(`YYYY-MM-DD_DayX.md`)를 생성할 때 따라야 할 구조와 프롬프트 가이드를 정의합니다.

## 1. 파일 생성 규칙 (File Generation Rules)
- **파일 이름 포맷**: `{YYYY-MM-DD}_Day{N}.md` (예: `2026-02-02_Day6.md`)
- **저장 위치**: `D:\unity\workspace\FC\Assets\MD\DevLog`
- **입력 필수 항목**: 개발 일차(Day), 주요 구현 항목, 영상 하이라이트 후보

## 2. 마크다운 템플릿 구조 (Markdown Structure)

CLI는 사용자의 입력을 받아 아래의 마크다운 포맷으로 출력해야 합니다.

---

# [Day {N}] {오늘의_핵심_작업_한줄_요약}

## 📅 기본 정보
- **날짜**: {YYYY-MM-DD} ({요일})
- **개발 일차**: {N}일차
- **작업 카테고리**: (선택: 매치엔진 / AI / UI / 데이터 / 아트 / 최적화)

## 🛠️ 오늘의 구현 (Implementation)
> 오늘 작업한 핵심 기능과 로직을 기술합니다.
- [ ] **{기능명 1}**: {상세 내용}
- [ ] **{기능명 2}**: {상세 내용}
- *기술적 메모*: (예: $P_{Goal}$ 공식 수정, NavMesh와 Rigidbody 동기화 문제 해결 등)

## 🎥 영상 소스 & 연출 (For YouTube)
> 1분 내외의 개발 로그 영상에 들어갈 시각적 요소를 기록합니다.
1. **메인 샷**: (예: 6vs6 경기 장면, 공이 휘어지는 궤적)
2. **버그/개그 샷**: (예: 키퍼가 공을 피해 도망감, 선수가 겹쳐서 날아감)
3. **강조할 점**: (예: "이제 태클하면 물리적으로 넘어집니다")

## 🐛 이슈 및 버그 (Issues)
- **해결됨**: {해결된 버그 내용}
- **진행 중**: {아직 고치지 못한 문제} (예: 3D 모델 애니메이션 싱크 안 맞음)

## 자동 완성 가이드 (Auto-Fill Guide)
*만약 사용자가 내용을 비워둘 경우, 기존 문서를 참고하여 다음 내용을 제안(Suggestion)할 수 있습니다.*

* **물리 공식 참조**: `3D_축구_매치_엔진_개발_문서화.md`의 $P_{Goal}$, $P_{Pass}$, 마그누스 효과 공식.
* **능력치 참조**: `선수-테이블.md`의 변수명 (예: `Stat_Strength`, `Stat_Tech`).
* **역할 참조**: `라이센스-단계-전술-지침.md`의 포지션 역할 (예: 인버티드 윙백, 하프백).
